# Scale Correction For Child Elements

Check example below. If you change the width or height of the parent, even with the `layout` keyword, it still affect the child serialize. To fix this, you have to also add `layout` in the child. In a sense, the child is like picking up the size change in parent, and respond accordingly.

One thing I find is if you use `animate` to width or height and the flex or grid layout, the child even without `layout`, it still change its size nicely. Also animate will push away the sibling element, so layout will change nicely.
<ScaleCorrection />

```js
// with both layout on child and parent
<div className={styles.exampleWrap}>
    <motion.div
        layout
        className={styles.item}
        style={{
            width: clicked ? 160 : 120,
            height: clicked ? 160 : 120,
        }}
        onClick={() => setClicked(!clicked)}
    >
        <motion.div layout className={styles.dot}/>
    </motion.div>

    <p>layout only on both parent & child</p>
</div>

// old way of animating elements
<div className={styles.exampleWrap}>
    <motion.div
        className={styles.item}
        animate={{
            width: clicked ? 160 : 120,
            height: clicked ? 160 : 120,
        }}
        onClick={() => setClicked(!clicked)}
    >
        <div className={styles.dot}/>
    </motion.div>
    <p>Just animate</p>
</div>
```
